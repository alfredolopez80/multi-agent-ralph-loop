# Halls — Rules Organized by Type

**Purpose**: Cross-topic organization by rule type (what kind of guidance is it?).

| File | Type | Description |
|------|------|-------------|
| `decisions.md` | Architectural decisions | ADR-style choices that were made and why |
| `patterns.md` | Positive patterns | Things that work well — use these |
| `anti-patterns.md` | Negative patterns | Things to avoid — these cause problems |
| `fixes.md` | Bug fixes & corrections | Specific bugs caught and how they were fixed |

## Usage

Load halls when you need type-based retrieval:
- "What patterns have we established?" → `patterns.md`
- "What decisions were made and why?" → `decisions.md`
- "What should I avoid?" → `anti-patterns.md`
- "What bugs were fixed in this domain?" → `fixes.md`

## Rooms (topic-based alternative)

See `../rooms/` for the same rules organized by topic instead of type.
